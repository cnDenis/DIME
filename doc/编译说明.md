# DIME 编译说明

本文说明如何在 Windows 上编译、部署与打包 **DIME（迪弥五笔输入法）**。

产物是 COM DLL（`dime.dll`），通过 TSF 注册后由系统加载。x64 与 x86 必须分别构建并分别注册，才能同时覆盖 64 位与 32 位应用。

---

## 1. 环境要求

| 依赖 | 说明 |
|------|------|
| Windows 10/11 | 开发与运行均需 Windows |
| Visual Studio 2022 | 需安装 **使用 C++ 的桌面开发**；脚本默认路径为 Community 版 |
| CMake | VS 自带即可；也可单独安装（≥ 3.20 推荐，工程最低 3.10） |
| Ninja（可选） | 使用 `CMakePresets.json` 的 Ninja 预设时需要 |
| Inno Setup 6（可选） | 仅打包安装程序时需要，默认路径 `C:\Program Files (x86)\Inno Setup 6\` |

脚本中的 VS 路径写死为 Community：

```text
C:\Program Files\Microsoft Visual Studio\2022\Community\...
```

若安装的是 Professional / Enterprise，请改 `scripts\build_release.bat` 与 `scripts\build_debug.bat` 里的 `CMAKE` / `MSBUILD` 路径，或改用下文的手动 CMake 命令。

---

## 2. 推荐流程一览

| 目的 | 命令 |
|------|------|
| 日常 Debug 编译（x64 + x86） | `scripts\build_debug.bat` |
| Release 编译（x64 + x86） | `scripts\build_release.bat` |
| 编译后注册到本机测试 | 管理员运行 `scripts\deploy_test.bat` |
| 一键出安装包 | `scripts\package.bat`（需 Inno Setup 6） |

在仓库根目录打开 **x64 Native Tools** 或普通命令行均可；`deploy_test.bat` 必须以管理员身份运行。

---

## 3. 用脚本编译（MSBuild）

脚本会用 VS 生成器配置工程（首次），再用 MSBuild 编译：

```bat
REM Debug
scripts\build_debug.bat

REM Release
scripts\build_release.bat
```

### 产物路径

| 架构 / 配置 | DLL | 词库工具 |
|-------------|-----|----------|
| x64 Debug | `out\x64\Debug\dime.dll` | `out\x64\Debug\build_bindict.exe` |
| x86 Debug | `out\x86\Debug\dime.dll` | （同解决方案一并生成） |
| x64 Release | `out\x64\Release\dime.dll` | `out\x64\Release\build_bindict.exe` |
| x86 Release | `out\x86\Release\dime.dll` | （同解决方案一并生成） |

中间工程目录：`out\x64\`、`out\x86\`（含 `dime.sln`）。

### 清理后重配

删除对应目录的 `CMakeCache.txt`（或整个 `out\x64` / `out\x86`）后重新跑脚本即可重新配置。

---

## 4. 用 CMake Presets 编译（Ninja）

预设定义在仓库根目录 `CMakePresets.json`。需已安装 Ninja，并在能调用到对应架构 `cl.exe` 的环境中执行（建议用「x64 Native Tools Command Prompt」或先 `vcvarsall.bat`）。

```bat
REM 配置
cmake --preset x64-debug
cmake --preset x86-debug
cmake --preset x64-release
cmake --preset x86-release

REM 构建
cmake --build --preset x64-debug
cmake --build --preset x86-debug
cmake --build --preset x64-release
cmake --build --preset x86-release
```

产物目录与脚本方式一致：`out\x64\Debug`、`out\x86\Debug`、`out\x64\Release`、`out\x86\Release`。

也可不用 preset，手动指定生成器：

```bat
cmake -S . -B out\x64 -A x64 -G "Visual Studio 17 2022"
cmake --build out\x64 --config Release

cmake -S . -B out\x86 -A Win32 -G "Visual Studio 17 2022"
cmake --build out\x86 --config Release
```

## 5. 本机部署测试

先完成 x64 + x86 的 Debug（或 Release）编译，再以**管理员**运行：

```bat
scripts\deploy_test.bat
```

脚本会：

1. 用 `build_bindict.exe` 从 `Dictionary\*.txt` 生成 `.bin`（若工具存在）
2. 将 DLL 复制到 `out\test\<序号>\x64\dime64.dll` 与 `...\x86\dime32.dll`
3. 码表放入同级共享目录 `out\test\<序号>\dict\`
4. 注销上一份测试部署，再分别用 `System32\regsvr32`（x64）与 `SysWOW64\regsvr32`（x86）注册

之后在系统输入法列表中启用 DIME；切换版本后建议重启记事本 / Word 等进程。

### 手动注册 / 注销

```bat
REM 安装（路径按实际 DLL 调整）
%SystemRoot%\System32\regsvr32.exe  "C:\Program Files\DIME\dime64.dll"
%SystemRoot%\SysWOW64\regsvr32.exe  "C:\Program Files\DIME\dime32.dll"

REM 卸载
%SystemRoot%\System32\regsvr32.exe /u "C:\Program Files\DIME\dime64.dll"
%SystemRoot%\SysWOW64\regsvr32.exe /u "C:\Program Files\DIME\dime32.dll"
```

两个 DLL 可同目录存放（同 CLSID、不同注册表视图互不冲突）。码表放在与 DLL 平级或引擎探测到的 `dict\` 目录，由 x64/x86 共用。

## 6. 打包安装程序

```bat
scripts\package.bat
```

步骤概要：

1. 调用 `build_release.bat` 编译 x64 + x86 Release  
2. 预编译 `Dictionary\wubi98.bin`、`pinyin.bin`  
3. 组装 `out\dist\DIME\`（`dime64.dll`、`dime32.dll`、`build_bindict.exe`、`dict\`）  
4. 用 Inno Setup 编译 `installer\DIME.iss`，生成安装包  

安装包输出：

```text
out\dist\DIME_<版本>_Setup.exe
```

### 版本号规则

- 若当前 HEAD 有精确匹配的 `v*` git tag：安装包版本取该 tag（去掉前缀 `v`）
- 否则：取 `src\Version.h` 中的 `DIME_VER_MAJOR.MINOR.PATCH`，再拼上当天日期 `yyyyMMdd`，例如 `1.2.1-20260719`

发布前改版本请编辑 `src\Version.h` 中的三个宏；该文件同时供 `DIME.rc` 版本资源与设置页「关于」使用。

## 7. 词库与 build_bindict

运行时优先加载 `dict\` 下的 `.bin`。文本码表为 UTF-16 LE，格式 `"编码"="词条"`。

手动编译示例：

```bat
out\x64\Release\build_bindict.exe Dictionary\wubi98.txt Dictionary\wubi98.bin
out\x64\Release\build_bindict.exe Dictionary\pinyin.txt Dictionary\pinyin.bin --max-code 24
```

若安装目录缺少合法 `.bin`，运行时会尝试调用同目录下的 `build_bindict.exe` 从 `.txt` 现场生成。二进制格式说明见 [BinaryDictionaryFormat.md](BinaryDictionaryFormat.md)。

## 8. 常见问题

**只编了 x64，32 位软件里没有输入法**  
必须再编并注册 x86 DLL。32 位进程只加载 WOW64 视图下注册的模块。

**脚本报找不到 cmake / MSBuild**  
确认已装 VS2022 C++ 工作负载；非 Community 版时修改脚本内路径，或改用手动 `cmake` / `cmake --preset`。

**deploy_test 提示需要管理员**  
注册 COM / 输入法配置需要提升权限，请用「以管理员身份运行」打开终端后再执行。

**package.bat 报找不到 Inno Setup**  
安装 [Inno Setup 6](https://jrsoftware.org/isinfo.php)，或修改脚本中的 `ISCC` 路径。

**改 DLL 后输入法行为没变**  
先注销旧路径、部署新文件再注册；并结束仍加载旧 DLL 的进程（或注销/重登）。`deploy_test.bat` 会尝试结束 `ctfmon.exe` 以减少占用。
